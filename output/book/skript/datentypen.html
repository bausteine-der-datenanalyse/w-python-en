2  Data Types

Objects that store data have a data type. The data type specifies how the stored values should be interpreted by Python. For example, the value “1” in Python can represent a character, an integer, a Boolean value, the month of January, the weekday Tuesday, or the level of a categorical variable.

In this section, the data types that are most important for data analysis are introduced.

2.1 Numbers

Numbers include integers, Boolean values (truth values), floating-point numbers, and complex numbers (which are not discussed further here).

Integers

Integers are entered in decimal notation by default and can be positive or negative.

print(12, -8)
12 -8

In addition, integers can also be specified in other bases:

  • Binary system: digits 0 and 1 with the prefix 0b
print(0b1000, "plus", 0b0000, "plus", 0b0010, \
      "plus", 0b0001, "is", 0b1011)
8 plus 0 plus 2 plus 1 is 11
  • Octal system: digits 0 to 7 with the prefix 0o
print(0o7000, "plus", 0o0700, "plus", 0o0020, \
      "plus", 0o0000, "is", 0o7720)
3584 plus 448 plus 16 plus 0 is 4048
  • Hexadecimal system: digits 0 to F with the prefix 0x
print(0xF000, "plus", 0x0200, "plus", 0x00A0, \
      "plus", 0x0001, "is", 0xF2A1)
61440 plus 512 plus 160 plus 1 is 62113

Floating-Point Numbers

Floating-point numbers are written either with the decimal point . or in exponential notation. Floating-point numbers have the type float.

print(120.6, 1206e-1, 12060e-2, "\n")

print("When representing lottery winnings in exponential notation, the sign matters.")
print(1e-3, "or", 1e+3)
120.6 120.6 120.6 

When representing lottery winnings in exponential notation, the sign matters.
0.001 or 1000.0

Since computers operate in the binary system, decimal numbers cannot be stored exactly. For example, the division of 1 by 10 in decimal is 0.1. In binary, 1₂ divided by 1010₂ is a repeating fraction:

\[ \frac{1_2}{1010_2} ~ {=} ~ 0,000\overline{1100}_2 \]

Therefore, decimal numbers must be approximated as a fraction of two integers (the binary fraction that approximates 0.1 is, in decimal notation, 3602879701896397 / 255). As a result, multiple floating-point numbers can be represented by the same binary approximation. Python outputs the shortest decimal representation, but since calculations are performed in binary, the nearest binary approximation—and therefore the corresponding shortest decimal—can change during computations (more information in the Python documentation).

print(0.1)  # The shortest decimal representation of the binary approximation
print(format(0.1, '.17g'))  # The next longer decimal representation of the same binary approximation
print(0.3 - 0.2)  # Computed in binary, the binary approximation may change
0.1
0.10000000000000001
0.09999999999999998

In practical work with Python, you may occasionally encounter results that seem unusual at first glance.

print(0.1 + 0.2)
print(0.01 + 0.02)
print(0.001 + 0.002)
print(0.0001 + 0.0002)
print(0.00001 + 0.00002)
0.30000000000000004
0.03
0.003
0.00030000000000000003
3.0000000000000004e-05

2.2 Arithmetic Operators

Arithmetic operators allow you to perform basic calculations. The result is usually of type float, except when both operands are of type int and the result can be represented as an integer.

Operator Description
+, - Addition / Subtraction
*, / Multiplication / Division
//, % Integer Division / Remainder
** Exponentiation

 

When combining multiple operators, their order must be respected or enforced using parentheses, e.g., (1 + 2) * 3. The same rules as in standard arithmetic apply. A complete overview of operator precedence is available in the Python documentation. For arithmetic operators, the precedence from highest to lowest is:

Operator
**
*, /, //, %
+, -

Operators of the same precedence are evaluated from left to right.

2.4 Boolean Values

The boolean values True and False are the result of logical expressions, which we will explore in more detail later. They also take the values 1 and 0 and therefore belong to the number type in Python.

print("Is 10 greater than 9?", 10 > 9)
print("Is 11 less than 10?", 11 < 10)
print("Is 10 exactly 10.0?", 10 == 10.0, "\n")

print("True and False can be added using +:", True + False)
print("... and multiplied using *:", True * False, "\n")
Is 10 greater than 9? True
Is 11 less than 10? False
Is 10 exactly 10.0? True 

True and False can be added using +: 1
... and multiplied using *: 0

Multiplying Boolean values is useful for combining multiple logical conditions into a logical AND:

print("Is 10 > 9 AND 10 > 8?", (10 > 9) * (10 > 8))
Is 10 > 9 AND 10 > 8? 1

The bool() function returns the truth value of a value.

print("Is 10 > 9 AND > 8?", bool((10 > 9) * (10 > 8)))
Is 10 > 9 AND > 8? True

Most values in Python have the truth value True.

print(bool(1), bool(2), bool(2.4))
print(bool('a'), bool('b'), bool('ab'))
True True True
True True True

Besides False and 0, empty or non-existent values or objects evaluate to False.

print(bool(False), bool(0))
print(bool("")) # an empty string
print(bool([])) # an empty list
print(bool(())) # an empty tuple
print(bool({})) # an empty dictionary
print(bool(None)) # None represents a non-existent value
False False
False
False
False
False
False

The collection types list, tuple, and dictionary will be introduced in the following chapters. Boolean values can control the execution of program code by acting like on and off. This way, code with an if statement is only executed if a collection type actually contains values.

my_list = ['Apples', 'Butter']
if my_list:
    print(f"We need to buy {my_list}.")

my_list = []  # an empty list
if my_list:
    print(f"We need to buy {my_list}.")
We need to buy ['Apples', 'Butter'].

2.5 Logical Operators

Logical operators include the logical connections and, or, and not. In addition, comparison operators such as >, >=, or == can also be used. The result of these operations is of type bool. Operators are executed in the following order. Operators of the same precedence are executed from left to right.

Operator Description
& bitwise AND
^ bitwise XOR
| bitwise OR
<, <=, >, >=, !=, == less than / less than or equal / greater than / greater than or equal / not equal / equal
not logical NOT
and logical AND
or logical OR
Hinweis 2.1: Bitwise Operators

Special caution is required with bitwise operators. These do not compare numbers as a whole, but rather bit by bit (in the binary system). Note that bitwise operators have higher precedence than comparison operations.

print(10 > 5 and 10 > 6)
print(10 > 5 & 10 > 6)
print(5 & 10)
print(10 > False > 6)
print((10 > 5) & (10 > 6))
True
False
0
False
True
Tipp

In general, bitwise operators are not needed for data analysis. Avoid unnecessary mistakes: do not use the bitwise operators &, ^, and |.

However, the operators &, ^, and | have a different meaning for sets (which we will cover later). In other modules, these operators also have a different syntactic role, e.g., in the Pandas package when passing multiple slicing conditions: df = df[(condition1) & (condition2) | (condition3)].

2.7 Strings

Strings are enclosed in single or double quotes in Python.

print('a string')
print("another string")
a string
another string

Within a string, single or double quotes can be used, as long as they don’t match the quotes surrounding the string.

print('A sophisticated heap beam, which we call a "LASER".')
print("I've turned the moon into what I like to call a 'Death Star'.")
A sophisticated heap beam, which we call a "LASER".
I've turned the moon into what I like to call a 'Death Star'.

The control character \ (also called the escape character) allows you to use certain special characters.

print("The control character \\ allows the same \"quotation marks\" in the output of print.")
print("First a\tTab, then a\nnew line.")
The control character \ allows the same "quotation marks" in the output of print.
First a Tab, then a
new line.

A prefixed r makes the backslash \ treated literally (raw string literal). This is useful, for example, when working with file paths.

print("The data is located at: C:\tolle_daten\nordpol\weihnachtsmann")
print(r"The data is located at: C:\tolle_daten\nordpol\weihnachtsmann")

2.10 Variables

Variables are placeholders or references to data. Assignment is done using the assignment operator =. A variable name may only consist of letters, numbers, and underscores. The first character cannot be a number.

var_1 = 'ABC'
var_2 = 26
var_3 = True

print("The", var_1, "has", var_2, "letters. This is", var_3)
The ABC has 26 letters. This is True

In Python, variables do not need to be initialized. The data type of a variable is determined by assigning an appropriate value.

print("The variable var_1 has the type", type(var_1))
print("The variable var_2 has the type", type(var_2))
print("The variable var_3 has the type", type(var_3))
The variable var_1 has the type <class 'str'>
The variable var_2 has the type <class 'int'>
The variable var_3 has the type <class 'bool'>

The data type of a variable changes when it is assigned a new value of a different data type.

var_1 = 100
print("The variable var_1 has the type", type(var_1))
The variable var_1 has the type <class 'int'>

Similarly, the data type of a variable can change to accommodate the result of an operation. In this example, the result cannot be stored as an integer. Python therefore assigns the data type ‘float’ to the object var_1.

var_1 = var_1 / 11
print("The variable var_1 has the type", type(var_1))
The variable var_1 has the type <class 'float'>

Python provides functions to determine the data type of a variable and to convert it. The function type() has already been used in the code examples to check the data type (i.e., class) of objects. The following code block demonstrates type conversion.

a = 67
print(a, type(a))

b = a + 1.8
print(b, type(b), "\n")

print(f"Note the truncation of the decimal part:\nConverting to integers with int(): {(a := int(a)), (b := int(b))}\n")

print(f"Converting to ASCII characters with chr(): {(a := chr(a)), (b := chr(b))}\n")

print(f"Converting to an ASCII number with ord(): {(a := ord(a)), (b := ord(b))}\n")

print(f"Converting to floating-point numbers with float(): {(a := float(a)), (b := float(b))}\n")

print(f"Converting to strings with str(): {(a := str(a)), (b := str(b))}\n")

print(f"Converting to Boolean values with bool(): {bool(a), bool(b)}")
67 <class 'int'>
68.8 <class 'float'> 

Note the truncation of the decimal part:
Converting to integers with int(): (67, 68)

Converting to ASCII characters with chr(): ('C', 'D')

Converting to an ASCII number with ord(): (67, 68)

Converting to floating-point numbers with float(): (67.0, 68.0)

Converting to strings with str(): ('67.0', '68.0')

Converting to Boolean values with bool(): (True, True)

Additional Assignment Operators

In the code block above, the so-called Walrus Operator := was used. This allows assignments to be made within an expression (here, inside the print() function). Python has a variety of other assignment operators (see the Python documentation or a more user-friendly overview here).

Operator Equivalent Assignment
a += 2 a = a + 2
a -= 2 a = a - 2
a *= 2 a = a * 2
a /= 2 a = a / 2
a %= 2 a = a % 2
a //= 2 a = a // 2
a **= 2 a = a ** 2
Tipp 2.4: Readable Assignments

The assignment operators shown in the table are easy to read and understand for someone reviewing your code, as assignments always appear at the start of a line, aligned to the left.

In contrast, the Walrus Operator can appear anywhere in your code. While this can be convenient while writing, it can reduce readability. If you use the Walrus Operator, make sure your code remains understandable.

Naming Variables

There are usually only a few rules for naming variables. Still, it is better to choose a long but descriptive variable name than a short one that is easy to type. Code is read far more often than it is written. Can you remember? What value is stored in the variable Var_3, and what values are stored in variable1 and a? It’s enough if you can recall the correct data type.

If you can’t remember, this is an example of poor naming: these variable names are not descriptive. The solution is in the expandable section below.

Tipp 2.5: Variable Solution 
print(var_3, type(var_3))
print(variable1, type(variable1))
print(a, type(a))
True <class 'bool'>
15 <class 'int'>
67.0 <class 'str'>

Therefore, it is recommended to use “descriptive,” i.e., self-explanatory, variable names. Self-explanatory variable names are those where the name describes the content of the variable. For example, if a variable stores a student’s degree, it can be named academic_degree or studienabschluss. If data from different years are being processed, the year can be included in the variable name for better distinction, e.g., academic_degree_2023 or studienabschluss2024. This improves code readability and makes using the variable easier. More information can be found in this Wikipedia section.

Hinweis 2.2: Keywords and Function Names

Reserved keywords and function names in Python are unsuitable as variable names. While Python will raise a syntax error if you try to assign a value to keywords like True or break, function names can be reassigned, for example with print = 6. After doing so, calling the print function will no longer work. In this case, you must remove the assignment from the script and restart Python.

The following keywords exist in Python:

assert     del        global     not        with
async      elif       if         or         yield
await      else       import     pass       True
break      except     in         raise      class
False      finally    is         return     None

2.11 Variable Exercises

  1. Write a script that converts a given time 𝗍 in seconds into the number of days, hours, minutes, and seconds, and prints this breakdown.
    Calculate the breakdown for the following times:

    1. 79,222 s

    2. 90,061 s

    3. 300,000 s

  2. The position of a vehicle at time \(t\), which accelerates constantly with acceleration \(a\), is given by:

\[ x(t) = x_0 + v_0 \cdot t + \frac{1}{2} \cdot a t^2 \]

Here, \(x_0\) is the initial position and \(v_0\) is the initial velocity. Create variables for \(x_0\), \(v_0\), and \(a\), and assign them values. Since the variables only store numerical values without units, consider any necessary unit conversions. You can use the following example values:

\[ x_0 = 10 \, km \,\,\,v_0 = 50 \frac{km}{h} \,\,\,a = 0.1 \frac{m}{s^2} \]

Create a variable for the time \(t\), e.g.: \(t = 10 \, min\).
Use the above equation and your variables to calculate the position \(x(t)\).
Output not only the value of \(x(t)\) in kilometers, but embed it in a full sentence including units.

def time_breakdown(t):
    days = t // (24*3600)
    t = t % (24*3600)

    t = t % 3600
    minutes = t // 60
    seconds = t % 60
    
    return days, hours, minutes, seconds
